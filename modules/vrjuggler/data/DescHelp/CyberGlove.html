Cyber Glove Chunk

Cyber Glove Chunk token: ``CyberGlove''
Provides configuration information for a CyberGlove device. Both hardware and software configuration can be done with this chunk. On the hardware side, the serial port to which the glove is connected and the baud rate at which data is sampled can be set. For the software configuration, the directory containing calibration data and the VR Juggler position proxy can be set.
Port Property token: ``port''
Sets the serial port to which the glove is connected. This is named by a string corresponding to the UNIX device node for the serial port on a UNIX system.
Baud Rate Property token: ``baud''
Sets the sampling rate for the serial port to which the CyberGlove is connected. This is an integer value that must be valid for the serial port and the glove.
Calibration Directory Property token: ``calDir''
Names the directory where calibration data for the CyberGlove can be found. This is a sting value, and as the name of a file, it depends upon the underlying file system for its syntax. On UNIX, file names are case sensitive, and directories are separated by the `/' character. On Win32-based systems, file names are not case sensitive, and directories are separated by the `\' character.
Glove Position Property token: ``glovePos''
Sets a position proxy for the CyberGlove position. This should point to the tracker that is on the glove. The proxy points to another chunk that must be in a predefined set of chunks. The current allowed chunk types are the following:
  • proxyAlias
  • PosProxy
Thus, the position proxy can point directly to a PosProxy chunk or to an alias for a PosProxy chunk. New types can be added (or changed) in the chunk description file (chunksDesc).

Cyber Glove chunk definition

chunk CyberGlove "CyberGlove" "Configuration info for the CyberGlove stuff."
   Name String 1 "Name" "Unique name of an instance of this chunk type"
   port String 1 "port" " serial port that the CyberGlove is connected to"
   baud Int 1 "baud" ""
   calDir String 1 "calibrationDir" "Home directory for the calibration data"
   glovePos Chunk 1 "glovePosition" vj_enumeration { "proxyAlias" "PosProxy" } "The position proxy for the glove position. i.e. the tracker on the glove."
   end
