VjControl command line arguments

vjcontrol [-n hostname[:port]] [-noautoload] [extensionfile.jar] [-o file] [configfile.config] [-d descfile.config]

Networking Options

  • -n hostname[:port] If a hostname and (optionally) port are specified, vjcontrol will try to establish a network connection to a running VR Juggler application on that machine. If no port number is given, the port specified in VjControl's preferences is used.

File Loading Options

  • -noautoload Vjcontrol can be configured to automatically load certain files at startup. This switch suppresses automatic loading, so that only files given on the command line are loaded.
  • -j name Loads VjControl extensions. If name is a .jar file, it loads that file. If name is a directory, it loads all .jar files in that directory.
  • -o name Loads the Chunk Organization tree name
  • -d name Loads the Chunk Description file name
  • -c name Loads the Configuration file name
  • -p name Loads a VR Juggler performance logging file
