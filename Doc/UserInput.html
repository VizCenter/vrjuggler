User Input Documentation

This document explains how user input and default input devices are handled by the C2Library.

Requirements

User must be able to

  • Ask for head
  • Ask for wand
  • Ask for other default devices
  • Ask for specific deviecs
  • Query the device for its data
The user will call a function like:

int index = GetDeviceIndex("Device Name");

The user will actually get an int index that can be used to call:

  • GetPos(int index)
  • GetAnalog(int index), etc.

    Proxy Aliases

    These are proxy names alias other proxies. The config file has a proxyAlias chunk  that defines an alias name to point to a proxy. These chunk sets up an array of indices to be used internally.

    The chunk looks like this:

      proxyAlias
        aliasName  string 1            # Name of the alias
        proxyPtr chunk 1               # Proxy to alias
        end
    Standard Aliases: These are suggested string aliases that should be used in all configs
    • C2Head - A positional device for the users head
    • C2Wand - A positional device for the user wand
    • C2Glove - A glove device.
    • C2Digital - A digital device
    • C2Analog - An analog device
    • C2Simulator - Positional device for controling simulator window
    These aliases anc then be used in the call to GetDeviceIndex("Device Name").  This call looks at the internal default device array.  This array contains the indices of the real proxies.  The function returns this proxy index.  This ability can be used to write programs that can be reconfigured to use other devices without code changes.
     

    Run-Time changes

    To change the configuration of proxy aliases at run-time, a proxyAlias chunk needs to be sent to the inputManager.  This will directly tell the input manager how to modify the alias array.
