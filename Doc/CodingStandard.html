Coding Standard

The following link has a good desciption of a coding standard and why one is needed:
http://www.possibility.com/Cpp/CppCodingStandard.html

The coding standard for VR Juggler will be based on a simplified version of the above standard.  The standard will be minimal and still try to allow for flexibility.  We just need a way to make the source code readably and maintainable across all classes.
 

Documentation

We will be using a modified PERCEPS script to automatically create documentation.  In order to allow for this, the source code must be commented with this in mind.
 

 Code Standard

The following is a brief summary of the way the code in VR Juggler should be written.  Of course there are always various exceptions, but if the code is relatively consistent it will make it much easier for us all.
 

Names

  • Class Names:  Name the class after what it does
  • Method Names: Name it after the actions it does
    • Suffixes are sometimes useful:
            Max - to mean the maximum value something can have.
            Cnt - the current count of a running count variable.
            Key - key value.
        For example: RetryMax to mean the maximum number of retries, RetryCnt to mean the current
        retry count.
    •     Prefixes are sometimes useful:
            Is - to ask a question about something. Whenever someone sees Is they will know it's a question.
            Get - get a value.
            Set - set a value.
