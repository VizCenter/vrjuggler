Digital Simulator Chunk

Sim Digital Chunk token: ``SimDigital''
Defines a simulated digital device. It uses a user-specified keyboard proxy to accomplish this simulation.
Keyboard Proxy Property token: ``keyboardProxy''
Sets the proxy pointing to the keyboard to use for the simulation. The proxy must point to a KeyboardProxy chunk. More chunk types can be added through the chunk description file (chunksDesc).
KeyPresses Defines a keypress to use to toggle a particular input. The number of Keypress lines determines the number of digital inputs supplied by this simulator. The user can add and remove keypress lines. Each keypress consists of two parts: a modifier and a key.

Modifier Keys
Lists the modifier keys that may be used together with normal keyboard keys and mouse buttons when increasing the value returned by the digital device. The modifier keys must be chosen from a predefined set of keys that are treated as modifiers. The current modifier keys are:

  • NONE (no modifier)
  • SHIFT (the <SHIFT> key)
  • CTRL (the <CTRL> or <CONTROL> key)
  • ALT (the <ALT> or <META> key)

Keys
The of keyboard key, mouse movement direction, or mouse button that can be used for toggling the value returned by the simulated digital device. All standard keys (except the F1-F12 keys), modifier keys (listed above), cursor movement keys, mouse movement directions and mouse buttons (for a three-button mouse) are available for use. Each key can be assigned to a particular aspect of the simulation, and modifiers can be used to allow the same keys (say those on the number pad) to be used for different aspects of simulation.

Sim Digital chunk definition

chunk SimDigital "Digital Simulator" "Simulated digital device.  Gets input from keyboard proxy."
  Name String 1 "Name" "Unique name of an instance of this chunk type"
  keyboardProxy Chunk 1 "Keyboard Proxy" vj_enumeration { "KeyboardProxy" } 
        "The keyboard to use for event input."
  keyPairs EmbeddedChunk -1 "KeyPresses" vj_enumeration { "KeyModPair" } 
        "Keypresses to trigger multiple digital inputs"
  end
