Intersense Tracker Device Chunk

The tracking system is composed of multiple stations connected to a single base unit; the base connects to a host computer via a serial interface. Configuration for the base (defined in the Chunk) includes serial port settings and a global transformation to be applied to all positional inputs.

Each station is a device such as a head sensor or wand. The station provides a single positional input, and may include an arbitrary number of digital and analog inputs.

The mapping of stations to the positional inputs of the Isense device is defined by the order the stations appear in the Stations property listed below. The mappings for digital and analog devices are specifically listed inside the configuration for each station; for example, the first station could provide digital units 4 through 7, while the second station provides units 0 through 3.

As with all input devices, positional, digital, and analog units are indexed separately (e.g. positional 0-3, digital 0-12, analog 0-2, etc.).
Intersense Tracker Device Chunk token: ``Isense''
Configuration for Intersense IS-9000 tracker.
Port Property token: ``port''
The string name of the serial port this device is attached to. On Unix systems, a typical value would be something like ``/dev/ttyd3''.
Baud Rate Property token: ``baud''
The speed of the serial port connection, in bps. Typical values might include 19200 and 38400.
Translation Property token: ``translate''
An XYZ translation to be applied to position data from this tracker.
Rotation Property token: ``rotate''
XYZ Rotation to be applied to position data from this tracker.
Stations Property token: ``stations''
Settings for the individual stations of this tracking system. Each value is a IsenseStation ConfigChunk.
Verbose Output Property token: ``verbose''
Enable output from the Intersense driver.
Startup script Property token: ``script''
File path of a script to be sent to the tracking system on startup.
Analog Lower Bound Property token: ``min''
Lower bound for analog data values. 0.0 is a reasonable value.
Analog Upper Bound Property token: ``max''
Upper bound for analog data values. 1.0 is a reasonable value.

Sim Position chunk definition

chunk Isense "Isense" "Intersense Tracking Devices"
  Name String 1 "Name" "Unique name of an instance of this chunk type"
  port String 1 "Port" "Serial port the Flock is attached to"
  baud Int 1 "Baud Rate" "Serial port speed"
  translate Float 3 "Translation" "(world_T_transmitter) Translation of 
                              the transmitter from Juggler coord system."
  rotate Float 3 "Rotation" "(world_T_transmitter) Rotation of the 
                              tranmitter from the Juggler coord system."
  stations EmbeddedChunk -1 "Stations" vj_enumeration { "ISenseStation" } 
                              "Settings for each Station."
  verbose Bool 1 "Vebose Enabled" "Enable output from the Intersense Driver"
  script String 1 "Script File" "File path of script sent to the tracker 
                              on startup."
  min Float 1 "min" "the actual analog data values range - lower bound"
  max Float 1 "max" "the actual analog data values range - upper bound"
  end
