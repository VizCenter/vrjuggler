Display System Chunk

Display System Chunk token: ``displaySystem''
Defines the graphics pipe setup on the machine. This chunk is used to set the number of pipes and the name for each pipe.
Num Pipes Property token: ``numpipes''
Tells VR Juggler how many pipes are on the system. This must be a non-negative integer value.
X Pipes Property token: ``xpipes''
Gives names for each of the graphics pipes on a system using X-Windows. Each string must be of the standard form used for naming an X display and screen (i.e., :display.screen). For example, the main display and screen is :0.0.

On a machine with two graphics pipes connected to one X display with two screens, the strings would be :0.0 and :0.1.

You can also use the value -1 instead of an X display name. In this case, the pipe will query the value of the user's $DISPLAY environment variable. This can be particularly useful with multiple users using multiple displays.

Currently, the number of pipes that can be named is set to eight (8), but this can be increased or decreased in the chunk description file (chunksDesc).

Display System chunk definition

chunk displaySystem "displaySystem" ""
   Name String 1 "Name" "Unique name of an instance of this chunk type"
   numpipes Int 1 "numpipes" "The number of pipes on the system"
   xpipes String 8 "xpipes" "The x-display strings for the pipes"
   end
