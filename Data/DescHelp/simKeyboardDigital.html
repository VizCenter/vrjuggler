Digital Keyboard Simulator Chunk

Digital Keyboard Simulator Chunk token: ``simKeyboardDigital''
This is a device that uses multiple digital devices to simulate a Keyboard (which can be used to drive other simulator devices, etc.).
simKeys Property token: ``simKeys''
Each simKeys line is used to define one simulated keypress. For each line, we give the keypress we're trying to simulate, a list of Digital Proxies that must be pressed to activate it, and a list of Digital Proxies that must not be pressed.

For example, if we have four wand buttons mapped to digital proxies, we could configure a Keyboard Simulator to give the keypress ``Y'' when buttons 1 and 2 are pressed and the others aren't, ``N'' when only buttons 3 and 4 are pressed, and ``?'' when the user presses all four buttons simultaneously.

Digital Keyboard Simulator chunk definition

chunk simKeyboardDigital "simKeyboardDigital" ""
  Name String 1 "Name" "Unique name of an instance of this chunk type"
  simKeys EmbeddedChunk -1 "simKeys" vj_enumeration { "simKeyDigital" } 
          "The keys we are simulated"
  end
