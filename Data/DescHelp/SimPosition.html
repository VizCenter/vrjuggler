Position Simulator Chunk

Sim Position Chunk token: ``SimPosition''
Defines a simulated position device. It uses a user-specified keyboard proxy to accomplish this simulation.
Keyboard Proxy Property token: ``keyboardProxy''
Sets the proxy pointing to the keyboard to use for the simulation. The proxy must point to a KeyboardProxy chunk. More chunk types can be added through the chunk description file (chunksDesc).
KeyPresses Defines the keypresses used to control this position simulator. This property has twelve values, allowing complete movement in six degrees of freedom.

Modifiers
The modifier keys must be chosen from a predefined set of keys that are treated as modifiers. The current modifier keys are:

  • NONE (no modifier)
  • SHIFT (the <SHIFT> key)
  • CTRL (the <CTRL> or <CONTROL> key)
  • ALT (the <ALT> or <META> key)

Keys
The keyboard key, mouse movement direction, or mouse button used to trigger a particular movement. All standard keys (except the F1-F12 keys), modifier keys (listed above), cursor movement keys, mouse movement directions and mouse buttons (for a three-button mouse) are available for use. Each key can be assigned to a particular aspect of the simulation, and modifiers can be used to allow the same keys (say those on the number pad) to be used for different aspects of simulation.

Initial Position Property token: ``initialPos''
Sets the initial position of the simulated device using three floating-point values.
Initial Rotation Property token: ``initialRot''
Sets the initial Euler rotation of the simulated device using three floating-point values giving the x, y and z angles.
Delta Translation Property token: ``dtrans''
The floating-point translation change made with each key press.
Delta Rotation Property token: ``drot''
The floating-point rotational change (in degrees) made with each key press.
Translation
Coordinate System 		Property token: ``transCoordSystem''
A boolean value (0 for false, 1 for true) stating whether or not the translations apply to the tracker's local coordinate system.
Rotation
Coordinate System 		Property token: ``rotCoordSystem''
A boolean value (0 for false, 1 for true) stating whether or not the rotations apply to the tracker's local coordinate system.

Sim Position chunk definition

chunk SimPosition "Position Simulator" "A simulated position device. Gets input from a keyboard proxy"
  Name String 1 "Name" "Unique name of an instance of this chunk type"
  keyboardProxy Chunk 1 "Keyboard Proxy" vj_enumeration { "KeyboardProxy" } 
           "The keyboard to use for event input."
  keyPairs EmbeddedChunk 12 "keyPairs" vj_valuelabels { "Forward" "Back" 
           "Left" "Right" "Up" "Down" "Rot Right" "Rot Left" "Rot Up" 
           "Rot Down" "Rot CCW" "Rot CW" } vj_enumeration { "KeyModPair" } 
           "Keys for moving and rotating the simulated device."
  initialPos Float 3 "Initial Position" vj_valuelabels { "X" "Y" "Z" } 
           "Initial position"
  initialRot Float 3 "Initial Rotation" vj_valuelabels { "X" "Y" "Z" } 
           "Initial Euler rotation"
  dtrans Float 1 "Translation Delta" "How much to translate for a single keypress."
  drot Float 1 "Rotation Delta" "How many degrees to rotate for a single keypress."
  transCoordSystem Int 1 "Translation Coordinate System" vj_enumeration { "Local=0" "Global=1" } 
           "Do the translations apply to the tracker's local coordinate system?  Ex. Wand - global, Camera - Local"
  rotCoordSystem Int 1 "Rotation Coordinate System" vj_enumeration { "Local=0" "Global=1" } 
           "Do the rotations apply to the tracker's local coordinate system?"
  end
