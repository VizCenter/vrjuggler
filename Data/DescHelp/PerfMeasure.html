Performance Measurements Chunk

Performance Measurements Chunk token: ``PerfMeasure''
A chunk that controls activation of VR Juggler performance measurements.
Timing Tests Property token: ``TimingTests''
This property can include a number of lines detailing whether certain performance tests should be active or inactive. Each line contains a string identifier and a boolean value - true to enable, false to disable.

The string identifiers are names of different classes of performance data. For example, the current beta includes a Juggler kernel test - ``Kernel'' and an OpenGL rendering pipe test - ``vjGlPipe''.

Any test in Juggler which does not have an entry in the Performance Measurements chunk (or any test at all if there is no chunk in the current configuration) defaults to being inactive.

The performance tests in the beta release are geared towards analyzing the performance of the Juggler library itself. The next release will feature tests better suited to analyzing performance issue of applications.

Performance Measurement chunk definition

chunk PerfMeasure "Performance Measurements" 
          "Controls performance monitoring of Juggler library"
  Name String 1 "Name" "Unique name of an instance of this chunk type"
  TimingTests EmbeddedChunk -1 "Timing Tests" vj_enumeration { "PerfTestActive" } 
          "Turn on/off Juggler internal timing tests"
  end


chunk PerfTestActive "PerfTestActive" "inner chunk for Performance Testing"
  Name String 1 "Name" "Unique name of an instance of this chunk type"
  Prefix String 1 "Prefix" ""
  Enabled Bool 1 "Enabled" ""
  end
