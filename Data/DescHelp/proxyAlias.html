Proxy Alias Chunk

Proxy Alias Chunk token: ``proxyAlias''
Defines an alias pointing to some proxy that will be used in user-level code. Only a known set of proxies are allowed to have aliases pointing to them.
Alias Name Property token: ``aliasName''
Gives the name of the alias to be used. Only one alias may be defined per proxy pointer.

WARNING!This property is deprecated, and is likely to be removed in a future release. For the time being, you should always use the same value for the Name and Alias Name properties.

Proxy Ptr Property token: ``proxyPtr''
The name of the proxy to which the alias refers. It specifies one of a predefined list of proxies that may be aliased. The pointer points to another chunk, and that chunk may be of the following types:
  • PosProxy
  • DigProxy
  • AnaProxy
  • GloveProxy
  • KeyboardProxy
  • GestureProxy
New allowable chunks must be added to the chunks description file (chunksDesc). The actual proxy being aliased is one in the list of proxies defined in the configuration file. Examples of such proxies could be the proxy for the head tracker, for the wand tracker or for a mouse (wand) button.

Proxy Alias chunk definition

chunk proxyAlias "proxyAlias" ""
   Name String 1 "Name" "Unique name of an instance of this chunk type"
   aliasName String 1 "aliasName" "The string alias to use."
   proxyPtr Chunk 1 "proxyPtr" vj_enumeration { "PosProxy" "DigProxy" "AnaProxy" "GloveProxy" "KeyboardProxy" "GestureProxy" } "Pointer to the proxy to which the alias refers."
   end
