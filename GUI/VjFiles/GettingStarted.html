Getting Started

Welcome to VR Juggler VjControl 1.0!

Setup

If you're reading this from the VjControl help menu, you've probably done this already, but if not, these are the steps that need to be taken before using VjControl:
  1. Set your VJ_BASE_DIR environment variable to the directory where VR Juggler is installed.
  2. Create a directory called .vjconfig in your home directory. This is where VjControl will store its own configuration information, and is a good general place to store all your Juggler-related configuration files.
  3. You should add the directory ${VJ_BASE_DIR}/bin to your PATH.
  4. Make sure that Java v1.1 is installed on your machine. Make sure that the directory where the java executable resides is also in your PATH.
  5. If you have been provided with a vjcontrol.cfg file, copy it into the ~/.vjconfig directory.

Starting VjControl

Simply type vjcontrol, and the VjControl program will start up without any files loaded. Once it's running, VjControl can be configured to load certain files at startup on subsequent executions. There are also command-line arguments that can be used to load files at startup.

Loading Files

VR Juggler uses two types of files: Config Files and ChunkDesc Files.
  • Config Files are made of 'Chunks' of information. For example, one chunk will describe the setup of a display device while another will describe the setup for a keyboard or tracking system. There will typically be several readymade config files in the $VJ_BASE_DIR/Data directory, with names like config.simulator or config.2wall.
  • ChunkDesc Files define the different kinds of chunks (ie. what kind of information a Keyboard chunk should have, or a Display chunk). The main ChunkDesc file for VR Juggler is usually $VJ_BASE_DIR/Data/chunksDesc.
To load a Config or ChunkDesc file, click on the "Configure" or "Descriptions" tabs of the VjControl GUI, and then press the "Load" button.
